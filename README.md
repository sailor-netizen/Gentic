## üöÄ How It Works

1. **Auto-Discovery**: The system automatically scans `.agent/skills/`. If you ask an agent to "deploy to staging," it loads `setup-ci-pipeline` and `deploy-staging`.
2. **Specialization**: Instead of one generic AI, you use specific personas defined in `agent.md`.
3. **Memory**: Agents check `project_memory.md` before making major changes to ensure they don't repeat past mistakes.

### 1. The "Self-Healing" CI/CD (Current Possibility)
**Scenario**: A build fails in CI.
**Workflow**:
1. **DevOps Agent** detects failure via `parse-logs`.
2. **Lead Developer** analyzes the error.

*Result*: Zero-touch bug fixing for minor issues.

## üìù Extending the System

To add a new skill:
1. Create a folder: `.agent/skills/my-new-skill`.
2. Add `SKILL.md` with:
   ```markdown
   ---
   description: Trigger phrase for this skill
   ---
   # Instructions
   Steps to perform the task...
   ```

To add a new agent:
1. Edit `.agent/agents/agent.md`.
2. Add a new entry with **Role** and **Prompt**.
